# Portfolio Repo Task List

This is the recommended order for improving the site with Codex in VS Code.

## Phase 1 — Personalize the content

- Replace placeholder email, GitHub, LinkedIn, and resume links.
- Update the hero headline and summary for the kinds of roles you want.
- Rewrite the About section using your actual background.
- Replace placeholder project descriptions with real projects.
- Update Experience so it connects RF/electronics troubleshooting to software engineering.
- Confirm the skills list matches what you can defend in interviews.

## Phase 2 — Strengthen the portfolio presentation

- Pick 1 featured project and expand it.
- Add stronger project cards with stack, purpose, and implementation details.
- Add screenshots, mockups, or polished placeholders.
- Add a resume CTA.
- Improve the contact section.

## Phase 3 — Polish the UI

- Improve spacing, typography, and section hierarchy.
- Make buttons and cards feel more premium.
- Improve mobile navigation and section scrolling.
- Check theme toggle behavior and visual consistency.
- Make sure the page feels strong on mobile widths first.

## Phase 4 — Quality pass

- Accessibility review.
- SEO/meta tag review.
- Check all links.
- Test on mobile and desktop sizes.
- Validate GitHub Pages compatibility.
- Push final version and enable Pages.

## Suggested first 5 Codex tasks

1. Rewrite the hero, About, and Experience sections for a mobile developer transitioning from electronics/RF troubleshooting.
2. Upgrade the visual design without adding frameworks or build tools.
3. Refactor the project section into stronger case-study style cards.
4. Add a resume CTA and improve the contact section.
5. Run an accessibility and responsive QA pass.

## Suggested repo hygiene

- Keep changes small and reviewable.
- Commit after each meaningful improvement.
- Use commit messages like:
  - `refine hero and summary copy`
  - `upgrade project card layout`
  - `add resume cta and contact polish`
  - `accessibility and responsive fixes`

## Preview checklist before publishing

- Does the hero immediately say what kind of developer you are?
- Are your best projects visible without too much scrolling?
- Can a recruiter find GitHub, LinkedIn, email, and resume in under 10 seconds?
- Does the site look clean on a phone?
- Are all external links working?
- Does the page still work as plain static HTML/CSS/JS?
